Daily Technical Snippet

What is late-interaction retrieval (ColBERT)?

Most vector search squashes a whole document into one embedding, so nuance gets averaged away. Late interaction (ColBERT) instead keeps a separate vector for every token, then scores a query against a passage by summing each query token's best match — matching at word granularity, not blob-vs-blob.

Learn more

ColBERT ("Contextualised Late Interaction over BERT") delays the query–document comparison until after both are encoded into per-token vectors. For each query token it finds its maximum-similarity token in the passage (a "MaxSim") and sums those — capturing that "did any part of this doc actually answer any part of my question?" far better than a single averaged vector, while staying cheap enough to run at scale via precomputed indexes. The trade-off is storage: you keep many small vectors per chunk instead of one, so teams pair it with quantisation or a cheap first-pass shortlist. For a URL-first memory layer like Kepler, late interaction is the difference between "roughly on-topic" recall and pulling the exact sentence that grounds an answer with a citation — precision is the product.

→ ColBERT: Efficient and Effective Passage Search (arXiv 2004.12832)
