Daily Technical Snippet

What is a temporal knowledge graph (for agent memory)?

Most memory systems store facts as if they were timeless — which is why an agent still thinks you're "going to Singapore" months after you got back. A temporal knowledge graph stores facts as edges between entities and stamps each one with when it was true and when it was recorded, so recall can reason over change instead of contradicting itself. In the 2026 memory benchmarks, temporal reasoning was the single biggest source of gains (+29.6 points).

Learn more

Flat vector memory answers "what's similar?" but not "what's true now?" A temporal (bi-temporal) knowledge graph models memory as a graph of entities and relationships — Alinta → supplies-gas-from → Chevron — where every edge carries two clocks: valid-time (when the fact held in the world) and transaction-time (when the system learned it). When a newer fact contradicts an old one, you don't overwrite — you close the old edge's validity window and open a new one, keeping an auditable history. Retrieval then combines semantic search with graph traversal as of a point in time. This is exactly the "fact-validity windows + contradiction handling" governed-memory rivals now market — and it's the natural spine for Kepler: a URL-first memory that can show you not just what it recalled but when that became true and what it replaced.
