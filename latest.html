Daily Technical Snippet

Reciprocal Rank Fusion — blending rankings without tuning weights

Real retrieval rarely rides one index. You run a keyword search, a vector search, maybe a graph query — and now you have three ranked lists that disagree. Reciprocal Rank Fusion (RRF) merges them using only each item's rank, not its raw score: every result scores 1/(k + rank) in each list (k a small constant, often 60), and you sum across lists. Items near the top of several lists float up — no score normalisation, no per-retriever weight tuning.

Learn more

RRF sidesteps the hardest part of hybrid search: BM25 scores, cosine similarities and graph relevances live on wildly different scales, so naively adding them lets one signal dominate. By throwing away magnitudes and keeping only ordinal rank, RRF is robust, near parameter-free, and easy to reason about — which is why it's the default fusion in Elasticsearch, OpenSearch and most production hybrid stacks. It also degrades gracefully: add or drop a retriever and the maths still holds. For Kepler the appeal is auditability — because fusion is just "which lists ranked this near the top", you can show why a memory surfaced (won on lexical and graph, lost on vector) instead of pointing at an opaque blended score. Explainable retrieval over a user-owned store beats a black-box relevance number.
