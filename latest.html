Daily Technical Snippet

Episodic vs semantic memory — the two things agents forget

Human memory comes in two flavours, and agents need both. Semantic memory is facts stripped of context (“Jacko founded Kepler”). Episodic memory is specific, timestamped events (“on 22 Jul the agent read doc-A and chose X”). Most vector stores blend everything into one soup of embeddings — so an agent can't tell a durable fact from a one-off event, and it either forgets what matters or over-remembers noise.

Learn more

The split comes from cognitive science and is formalised for agents in the CoALA framework, which divides an agent's memory into working, episodic, semantic and procedural stores — each written and read differently. Semantic memory wants deduplication and canonical facts; episodic memory wants an ordered, immutable log you can replay; procedural memory holds how to do a task. Collapsing them into one index is why naive RAG both hallucinates stale “facts” and loses the thread of what just happened. For Kepler this is the whole design argument: episodic memory is where auditability actually lives — a URL-addressable, timestamped event trail of exactly what the agent knew and did at each step, sitting cleanly alongside the durable semantic facts, so “why did it do that?” has an answer you can open.

→ Sumers et al., “Cognitive Architectures for Language Agents (CoALA)” (arXiv)
