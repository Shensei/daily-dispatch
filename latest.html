Daily Technical Snippet

What are vector embeddings?

A vector embedding turns a piece of text (or an image, or a URL's contents) into a list of numbers that captures its meaning, so "similar" things sit close together in a high-dimensional space. It's the trick that lets a system retrieve the right note by meaning rather than exact keyword — the backbone of every AI memory or "second brain" product, Kepler included.

Learn more

An embedding model maps input to a fixed-length vector (say 768 or 1,536 dimensions). Because semantically related items land near each other, you can measure closeness with cosine similarity and pull the top matches fast using a vector database (FAISS, pgvector, Pinecone). This is what powers retrieval-augmented generation: instead of stuffing everything into the prompt, you embed the user's query, fetch the nearest stored chunks, and feed only those to the model. For a URL-first memory product, the quality of your chunking + embedding strategy is the product — it decides whether the right context surfaces when it matters.
